What is PumpkinGuard?

PumpkinGuard is a zero-overhead anti-cheat and region protection plugin for Pumpkin MC servers. Designed from the ground up in Rust, it leverages the borrow checker to guarantee memory safety without a garbage collector — meaning no GC pauses, no unpredictable latency spikes, and no false positives during peak load.

The plugin hooks into Pumpkin's event system at compile time via procedural macros, achieving tick-level monitoring without allocating on the hot path.

Features

  • Real-time movement validation with configurable tolerance windows
  • Region protection with ACL-based permission trees
  • Async event processing — server tick is never blocked
  • Built-in integration with AuthForge for identity-bound protections
  • Web dashboard API (JSON over HTTP) for remote administration
  • Zero unsafe blocks — verified by cargo-geiger

Configuration

Drop a pumpkin_guard.toml in your server root. All configuration is hot-reloadable via SIGHUP or /guard reload:

[protection]
enabled = true
tolerance_ms = 120
max_velocity = 0.5

[regions]
default_acl = "owner-only"
persist_backend = "sqlite"   # or "postgres"

[monitoring]
log_level = "warn"
webhook_url = "https://your.discord.webhook"

Performance Benchmarks

Tested on a 500-player stress scenario with synthetic movement injection at 1,000 events/sec per player:

  • Average event latency: 0.08ms
  • P99 latency: 0.31ms
  • Memory overhead: ~4MB RSS per 100 active players
  • CPU usage: <2% on a single core (AMD EPYC 9754)
