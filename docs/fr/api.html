API Reference

Documentation de l'API REST de Pumpkin Hub.

API en développement
L'API est en Phase 1. La base de données, l'endpoint /health et l'authentification GitHub OAuth sont implémentés. Les endpoints ci-dessous marqués Planifié seront ajoutés dans les prochaines phases.

URL de base

http://localhost:8080/api/v1

Toutes les routes sont préfixées par /api/v1. Le versioning est géré par le chemin d'URL.

Headers communs

HeaderValeurDescription
Content-Typeapplication/jsonFormat des corps de requête/réponse
x-request-idauto-généréID unique de requête (traçabilité)

Health Check

GET /api/v1/health

Vérifie que l'API est opérationnelle et que la base de données est accessible. Utilisé par Docker pour les health checks.

Réponse 200 OK

{
  "status": "ok",
  "service": "pumpkin-hub-api",
  "version": "0.1.0",
  "database": "connected"
}

Réponse 503 Service Unavailable

{
  "status": "degraded",
  "service": "pumpkin-hub-api",
  "version": "0.1.0",
  "database": "disconnected"
}
ChampTypeDescription
statusstringÉtat du service : "ok" ou "degraded"
servicestringNom du service
versionstringVersion du crate (depuis Cargo.toml)
databasestringConnectivité DB : "connected" ou "disconnected"

Exemple cURL

curl -s http://localhost:8080/api/v1/health | jq

Gestion d'erreurs

L'API retourne des erreurs structurées en JSON via le type centralisé AppError. Chaque erreur inclut un code HTTP approprié et un message descriptif.

Format d'erreur

{
  "error": {
    "code": 404,
    "message": "Plugin not found"
  }
}

Codes d'erreur

CodeSignificationCause typique
400Bad RequestCorps de requête invalide, paramètres manquants
401UnauthorizedToken manquant ou expiré
403ForbiddenPermissions insuffisantes
404Not FoundRessource inexistante
422Unprocessable EntityValidation échouée
500Internal Server ErrorErreur serveur inattendue

Authentification

L'authentification repose sur GitHub OAuth 2.0. Après le flow OAuth, un JWT est stocké dans un cookie HttpOnly (SameSite=Lax). Les endpoints protégés acceptent le token via cookie ou via le header Authorization: Bearer <jwt>.

Protection CSRF
Un état CSRF à durée de vie courte est stocké dans un cookie HttpOnly lors du login et validé au callback, empêchant les attaques CSRF sur le flow OAuth.

GET   /api/v1/auth/github

Redirige l'utilisateur vers la page d'autorisation OAuth de GitHub. Scopes demandés : read:user, user:email.

Réponse

302 Found — redirection vers https://github.com/login/oauth/authorize?...

GET   /api/v1/auth/github/callback

Gère le callback OAuth de GitHub. Échange le code d'autorisation contre un access token, récupère le profil GitHub, fait un upsert en base et émet un cookie JWT.

Paramètres query

ParamètreTypeDescription
codestringCode d'autorisation fourni par GitHub
statestringCSRF state à valider contre le cookie

Réponse

303 See Other — redirige vers le frontend (première valeur de ALLOWED_ORIGINS) et pose le cookie pumpkin_hub_token.

Cas d'erreur

CodeCause
401Cookie CSRF absent ou state invalide
500Erreur lors de l'échange de token GitHub ou accès BDD

GET   /api/v1/auth/me   Protégé

Retourne le profil de l'utilisateur authentifié. Nécessite un JWT valide (cookie ou header Bearer).

Réponse 200 OK

  "username": "octocat",
  "display_name": "The Octocat",
  "email": "octocat@github.com",
  "avatar_url": "https://avatars.githubusercontent.com/u/1",
  "bio": null,
  "role": "author",
  "created_at": "2026-03-06T12:00:00Z"
}

display_namestring | nullNom d'affichage GitHub

GET   /api/v1/auth/logout

Supprime le cookie JWT (pumpkin_hub_token), déconnectant l'utilisateur.

Réponse 200 OK

{ "message": "Logged out" }

Claims JWT

Le JWT est signé avec HS256. Le secret est configuré via JWT_SECRET. La durée de vie par défaut est 24 heures (JWT_TTL_SECONDS=86400).

ClaimTypeDescription
subUUIDIdentifiant interne de l'utilisateur
usernamestringNom d'utilisateur GitHub à l'émission
rolestringRôle à l'émission
iatUnix timestampDate d'émission
expUnix timestampDate d'expiration
