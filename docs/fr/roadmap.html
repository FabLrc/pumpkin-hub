Roadmap

Planification des fonctionnalités et évolution de Pumpkin Hub.

Vision

Pumpkin Hub vise à devenir la plateforme de référence pour l'écosystème Pumpkin MC — un registre de plugins performant, sécurisé et communautaire, nativement pensé pour Rust. La roadmap ci-dessous détaille les grandes phases de développement pour atteindre cet objectif.

Légende
Terminé En cours Planifié Backlog

Phase 0 — Fondations Terminé

Infrastructure de développement
  • Docker Compose multi-services (API, Frontend, PostgreSQL, Meilisearch)
  • Hot-reload avec cargo-watch (API) et next dev (Frontend)
  • Health checks et volumes persistés
Squelette API Rust
  • Architecture Axum avec build_app() testable
  • Middleware stack complet (CORS, Trace, RequestId, Timeout, Compression)
  • Gestion de configuration typée depuis les variables d'environnement
  • Type d'erreur centralisé AppError avec mapping HTTP automatique
  • Endpoint /api/v1/health opérationnel
Squelette Frontend
  • Next.js 16 avec App Router et React 19
  • Tailwind CSS 4 et configuration TypeScript
  • ESLint avec configuration Next.js
CI/CD et qualité
  • Pipeline GitHub Actions (lint, clippy, type-check, tests, build)
  • Cache NPM et Rust pour builds accélérés
Design System & Maquettes
  • Direction artistique "Brutaliste Industrielle" validée
  • 4 maquettes HTML haute-fidélité (Landing, Explorer, Plugin, Dashboard)
  • Typographies Raleway + JetBrains Mono, palette orange/noir

Phase 1 — Core Platform En cours

Base de données & Modèles
  • Intégration SQLx avec PostgreSQL (migrations, connection pool)
  • Modèles de données : Users, Plugins, Versions, Categories
  • Relations et contraintes d'intégrité
  • Seeds de développement avec données de test
Authentification GitHub OAuth
  • Flow OAuth 2.0 avec GitHub (login/callback)
  • Session management avec JWT ou cookies sécurisés
  • Middleware d'authentification pour les routes protégées
  • Profils utilisateurs synchronisés depuis GitHub
CRUD Plugins
  • Endpoints REST complets : list, create, read, update, delete
  • Validation des entrées (nom, description, catégorie, licence)
  • Pagination, tri et filtrage par catégorie
  • Slug unique et URL-safe pour chaque plugin
Implémentation des pages Frontend
  • Intégration du design system (composants React depuis les maquettes)
  • Page Landing avec trending plugins et statistiques globales
  • Page Explorer avec recherche et filtres
  • Page Plugin Detail avec informations et versions
  • Connexion à l'API via fetch/SWR

Phase 2 — Distribution Planifié

Gestion des versions & binaires
  • Upload de versions avec métadonnées (semver, changelog, compatibilité Pumpkin)
  • Support multi-architectures CPU (x86_64, aarch64)
  • Stockage des binaires sur Cloudflare R2
  • Génération automatique de checksums SHA-256
  • Vérification d'intégrité au téléchargement
Recherche full-text
  • Indexation des plugins dans Meilisearch
  • Recherche instantanée avec typo-tolerance
  • Filtres facettés (catégorie, architecture, version Pumpkin)
  • Tri par pertinence, téléchargements, date
  • Suggestions et autocomplétion
Dependency Graph
  • Déclaration des dépendances entre plugins
  • Résolution de compatibilité (version ranges)
  • Visualisation graphique des dépendances
  • Alertes de conflits potentiels

Phase 3 — Communauté Planifié

Dashboard auteur
  • Tableau de bord avec KPIs : téléchargements, utilisateurs actifs, tendances
  • Gestion des plugins publiés (édition, nouvelles versions, dépublication)
  • Graphiques de téléchargements dans le temps
  • Gestion des clés API pour l'intégration CI/CD
Publication depuis GitHub
  • GitHub App pour lier un dépôt à un plugin
  • Publication automatique sur tag/release GitHub
  • Webhook pour synchroniser les README et changelogs
  • Badge dynamique pour afficher la version sur le README GitHub
Système de reviews
  • Notes et commentaires sur les plugins
  • Système de rating (étoiles + commentaire)
  • Modération par les auteurs et les administrateurs
  • Signalement de contenu abusif
Collections & Curations
  • Collections thématiques de plugins (ex: "Essential PvP Pack")
  • Mise en avant de plugins par l'équipe (Staff Picks)
  • Catégories éditoriales et tags communautaires

Phase 4 — Écosystème Planifié

CLI Pumpkin Hub
  • Outil CLI Rust pour publier, rechercher et installer des plugins
  • Commandes : ph publish, ph search, ph install
  • Fichier de lock pour la résolution de dépendances
  • Authentification via token API
API publique & Webhooks
  • Documentation OpenAPI/Swagger auto-générée
  • Rate limiting et quotas par clé API
  • Webhooks configurables (nouvelle version, nouveau plugin, téléchargement)
  • SDK clients pour intégrations tierces
Sécurité avancée
  • Scan automatique des binaires uploadés (antivirus, analyse statique)
  • Signature cryptographique des releases (GPG / cosign)
  • Audit log de toutes les opérations de publication
  • Programme de bug bounty communautaire
Internationalisation
  • Interface multilingue (EN, FR, DE, ES, PT, ZH)
  • Descriptions de plugins multilingues
  • Documentation traduite

Backlog — Idées futures

Ces fonctionnalités ne sont pas encore planifiées mais font partie de la vision long terme du projet. Elles seront priorisées en fonction des retours de la communauté.

Marketplace Monétisée
Support de plugins premium avec système de paiement intégré pour les auteurs.
Serveur de test cloud
Environnement Pumpkin MC éphémère pour tester un plugin directement depuis le navigateur.
Plugin Templates
Scaffolding de projets plugin via cargo generate avec templates officiels.
Analytics avancés
Métriques détaillées : rétention, versions actives, distribution géographique.
Système de rôles
Organisations avec membres, permissions granulaires et co-ownership de plugins.
API GraphQL
Endpoint GraphQL alternatif pour des requêtes flexibles côté client.
