Démarrage rapide

Installer et lancer l'environnement de développement Pumpkin Hub en quelques minutes.

Prérequis

OutilVersion minimaleVérification
Docker24.xdocker --version
Docker Compose2.20+docker compose version
Git2.xgit --version
Optionnel
Pour le développement local sans Docker : Rust 1.75+, Node.js 20+ et PostgreSQL 16.

Installation

# Cloner le dépôt
git clone https://github.com/FabLrc/pumpkin-hub.git
cd pumpkin-hub

Lancement avec Docker Recommandé

# Construire et lancer tous les services
docker compose up --build

Cette commande lance 4 services :

ServicePortDescription
frontend3000Next.js dev server avec hot-reload
api8080API Rust/Axum avec cargo-watch
postgres5432Base de données PostgreSQL 16
meilisearch7700Moteur de recherche full-text
Hot-reload
L'API et le frontend sont configurés en mode développement avec rechargement automatique à chaque modification de fichier source.

Lancement local (sans Docker)

API (Rust)

cd api
cargo install cargo-watch
cargo watch -x run

Frontend (Next.js)

cd frontend
npm install
npm run dev

Variables d'environnement

La configuration de l'API est gérée par des variables d'environnement. Les valeurs par défaut sont définies dans docker-compose.yml :

VariableDéfautDescription
HOST0.0.0.0Adresse d'écoute de l'API
PORT8080Port de l'API
DATABASE_URLpostgres://...Chaîne de connexion PostgreSQL
MEILISEARCH_URLhttp://meilisearch:7700URL du serveur Meilisearch
MEILISEARCH_KEYdev_master_keyClé master Meilisearch
RUST_LOGdebugNiveau de log (trace, debug, info)

Vérification

Une fois les services lancés, vérifier que tout fonctionne :

# Health check API
curl http://localhost:8080/api/v1/health

# Réponse attendue :
# {"status":"ok","service":"pumpkin-hub-api","version":"0.1.0","database":"connected"}
# Frontend
# Ouvrir http://localhost:3000 dans le navigateur

# Meilisearch
curl http://localhost:7700/health

# Réponse attendue :
# {"status":"available"}
Dépannage
Si un service ne démarre pas, vérifier les logs avec docker compose logs {service}. Les causes fréquentes sont un port déjà occupé ou un volume corrompu (docker compose down -v pour réinitialiser).
