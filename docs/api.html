API Reference

Documentation technique de l'API REST Pumpkin Hub (Axum / Rust).

Work in progress
L'API est en phase initiale. Seul l'endpoint /health est actuellement implémenté. Les endpoints listés "à venir" sont planifiés dans la roadmap.

Base URL

http://localhost:8080/api/v1

Tous les endpoints sont préfixés par /api/v1. Le versioning est géré par nesting dans le routeur Axum, ce qui permet d'ajouter un /api/v2 sans impact sur les routes existantes.

Headers communs

HeaderDirectionDescription
Content-TypeRequest / Responseapplication/json
x-request-idResponseUUID unique attribué à chaque requête (traçabilité)
Content-EncodingResponsegzip si le client supporte la compression
AuthorizationRequestBearer token (pour les endpoints authentifiés, à venir)

Health Check

GET   /api/v1/health

Retourne le statut de liveness du service. Utilisé par les healthchecks Docker et les load balancers.

Requête

curl -X GET http://localhost:8080/api/v1/health

Réponse 200 OK

{
  "status": "ok",
  "service": "pumpkin-hub-api",
  "version": "0.1.0"
}
ChampTypeDescription
statusstringÉtat du service ("ok")
servicestringNom du service
versionstringVersion du crate (depuis Cargo.toml)

Gestion d'erreurs

Toutes les erreurs sont retournées au format JSON unifié, via le type centralisé AppError : 

{
  "error": "Description de l'erreur"
}

Codes d'erreur

HTTP StatusVarianteSignification
401UnauthorizedAuthentification requise ou token invalide
404NotFoundRessource introuvable
422UnprocessableEntityDonnées invalides (détail dans le message)
500InternalErreur interne (détails loggés côté serveur, jamais exposés au client)
504TimeoutLa requête a dépassé le timeout global de 30 secondes
Sécurité
Les erreurs internes (500) ne transmettent jamais la cause racine au client. Le message est remplacé par "An unexpected error occurred" et la raison technique est tracée côté serveur via tracing::error!.

Politique CORS

La politique CORS est configurée via la variable d'environnement ALLOWED_ORIGINS (valeurs séparées par virgules).

ParamètreValeur
Origines autoriséesConfigurable via ALLOWED_ORIGINS
MéthodesGET, POST, PUT, PATCH, DELETE
Headers autorisésContent-Type, Authorization
CredentialsActivé (allow_credentials: true)
Max Age (preflight)3600 secondes (1 heure)
