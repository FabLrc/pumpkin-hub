Démarrage rapide

Installer et lancer Pumpkin Hub en environnement de développement.

Prérequis

OutilVersion minimaleUsage
Docker + Docker Compose24.x / v2Orchestration des services
RustEdition 2021 (1.70+)Compilation de l'API
Node.js20.xBuild du frontend
Git2.xContrôle de version

Installation

Cloner le dépôt et se positionner dans le répertoire :

git clone https://github.com/FabLrc/pumpkin-hub.git
cd pumpkin-hub

Lancement avec Docker Compose

La méthode recommandée pour le développement. Docker Compose orchestre 4 services :

  • frontend-dev — Next.js avec hot-reload (port 3000)
  • api-dev — API Rust avec cargo-watch (port 8080)
  • db-dev — PostgreSQL 16 Alpine (port 5432)
  • meilisearch-dev — Meilisearch v1.7 (port 7700)
# Construire et démarrer tous les services
docker compose up --build

# Ou en arrière-plan
docker compose up --build -d

# Voir les logs en temps réel
docker compose logs -f api-dev
Note
L'API attend que PostgreSQL et Meilisearch soient healthy avant de démarrer, grâce aux healthcheck configurés dans le compose.

Lancement local (sans Docker)

Pour développer un service individuellement, il est possible de le lancer hors Docker :

API Rust

cd api

# Vérifier la compilation
cargo check

# Lancer en mode développement
cargo run

# Avec hot-reload (nécessite cargo-watch)
cargo install cargo-watch
cargo watch -x run

Frontend Next.js

cd frontend

# Installer les dépendances
npm install

# Lancer le serveur de développement
npm run dev

# Vérifier le linting
npm run lint

# Build de production
npm run build

Variables d'environnement

L'API charge automatiquement un fichier .env à la racine de api/ via dotenvy. En mode Docker, les variables sont injectées par le compose.

VariableRequisDéfautDescription
DATABASE_URLOui—URL de connexion PostgreSQL
MEILISEARCH_URLOui—URL du serveur Meilisearch
MEILISEARCH_KEYOui—Clé maître Meilisearch
SERVER_HOSTNon0.0.0.0Adresse IP de bind
SERVER_PORTNon8080Port d'écoute
ALLOWED_ORIGINSNonhttp://localhost:3000Origines CORS (séparées par virgules)
RUST_LOGNonpumpkin_hub_api=infoNiveau de verbosité des logs
NEXT_PUBLIC_API_URLNon—URL de l'API côté frontend

Exemple de fichier .env

DATABASE_URL=postgres://pumpkin_user:dev_password@localhost:5432/pumpkin_hub_dev
MEILISEARCH_URL=http://localhost:7700
MEILISEARCH_KEY=dev_master_key
SERVER_PORT=8080
ALLOWED_ORIGINS=http://localhost:3000
RUST_LOG=pumpkin_hub_api=debug,tower_http=debug

Vérification de l'installation

Une fois les services démarrés, vérifier que l'API répond :

curl http://localhost:8080/api/v1/health

Réponse attendue :

{
  "status": "ok",
  "service": "pumpkin-hub-api",
  "version": "0.1.0"
}
CI/CD
Le pipeline d'intégration continue (GitHub Actions) exécute automatiquement sur chaque PR : lint + type-check + build pour le frontend, et cargo fmt + cargo clippy + cargo test pour l'API.
